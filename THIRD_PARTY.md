# Third-party provenance

This project is Apache-2.0. It vendors third-party source kept under its
original license and copyright headers (unmodified). This file records what is
vendored and why.

## Vendored components

- **NVIDIA TensorRT API headers** (`third_party/tensorrt-headers/include/Nv*.h`)
  - Source: https://github.com/NVIDIA/TensorRT, tag `v10.7.0` (`include/`)
  - License: Apache-2.0 (the open-source TensorRT headers)
  - The shim is source- and ABI-compatible with what consumer apps include;
    headers are vendored verbatim with their NVIDIA Apache-2.0 banners intact.
    No AMD copyright line is added to these files.

- **ONNX-TensorRT parser header** (`third_party/tensorrt-headers/include/NvOnnxParser.h`)
  - Source: https://github.com/onnx/onnx-tensorrt, branch `10.7-GA`
  - License: Apache-2.0

- **TensorRT sample + common helpers** (`vendor/tensorrt-samples/`)
  - Source: https://github.com/NVIDIA/TensorRT, tag `v10.7.0`
    (`samples/sampleOnnxMNIST`, `samples/common`, `samples/utils`)
  - License: Apache-2.0
  - Vendored UNMODIFIED and built against the shim, so correctness is judged by
    NVIDIA's own sample code rather than a bespoke harness. Their CUDA-runtime
    calls resolve to HIP via `include/compat`.

## Test assets

- `test/mnist/mnist.onnx` is the classic ONNX MNIST model (inputs `Input3`,
  output `Plus214_Output_0`) from the ONNX model zoo
  (https://github.com/onnx/models, MIT-licensed model), used by the stock
  sample.
- `test/mnist/{0..9}.pgm` are generated by `tools/gen_mnist_pgms.py` from the
  public MNIST test set; each is the most confidently classified test image for
  its digit (inverted per the sample's normalization). MNIST is a public
  research dataset (CC BY-SA 3.0).
- `test/models/mnist_cnn.onnx` + `test/golden/` are generated locally by
  `tools/make_model_and_golden.py` (deterministic, seeded); not third-party.
  Goldens come from the ONNX `ReferenceEvaluator` (shipped with `onnx`,
  Apache-2.0).
